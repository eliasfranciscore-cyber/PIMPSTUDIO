# Skill: Pendientes y Seguimiento (ELIJA)

## Propósito
Mantener un sistema de tareas confiable: priorizado, actualizado y conectado con objetivos de negocio.

## Cuándo usar este skill
- El usuario pide revisar pendientes del día o semana.
- Se necesita ordenar una lista larga de tareas.
- Se requiere seguimiento de tareas bloqueadas o atrasadas.

## Entradas esperadas
- Lista de tareas abiertas.
- Estado actual (pendiente, en curso, bloqueada, completada).
- Fecha objetivo y responsable (si existe).
- Prioridades de negocio vigentes.

## Flujo de trabajo
1. Normalizar tareas con formato estándar.
2. Priorizar por impacto y urgencia (`P1`, `P2`, `P3`).
3. Identificar bloqueadas y definir plan de desbloqueo.
4. Proponer plan diario y semanal.
5. Cerrar con tablero de seguimiento.

## Formato estándar de tarea
- `Tarea`: descripción clara.
- `Prioridad`: `P1`/`P2`/`P3`.
- `Responsable`: nombre.
- `Fecha objetivo`: fecha concreta.
- `Estado`: pendiente/en curso/bloqueada/completada.
- `Siguiente acción`: paso mínimo siguiente.

## Formato de salida recomendado
1. Top 3 tareas críticas.
2. Lista de bloqueos y acción de desbloqueo.
3. Plan de ejecución del día.
4. Riesgos por atraso + mitigación.

## Criterios de calidad
- Cada tarea debe tener dueño y fecha.
- Evitar tareas ambiguas (“revisar tema”).
- Mantener foco en progreso real diario.
- Marcar explícitamente dependencias externas.

## Notas operativas
- Si una tarea requiere coordinación, derivar al skill de correo.
- Si una tarea depende de reunión, derivar al skill de agenda.
- Si una tarea cierra oportunidad comercial, registrar aprendizaje en `knowledge/empresa.md`.
